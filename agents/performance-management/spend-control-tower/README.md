# Spend Control Tower

The **Spend Control Tower** agent watches **requisition-to-pay** for control breakdowns: **maverick spend**, **catalog bypass**, **approval path oddities**, and **policy drift**—then suggests **operational fixes**, not only charts.

## Why it matters

Savings and compliance erode quietly through **process leakage**. A control tower narrative helps CFO and COO stakeholders see **where the guardrails failed** and **how to fix the plumbing**.

## Common inputs

- P2P transactions across req → PO → receipt → invoice
- Policy definitions: who can buy what, from whom, with which approvals
- Org data for ownership routing

## Typical outputs

- Exception summary by dimension (category, region, supplier, user segment anonymized)
- Remediation plays: catalog, training, rule change, supplier enablement
- Weekly briefing storyline for ops cadence

## Workflow in practice

Run **weekly** ops reviews and **monthly** exec summaries. Pair with **SpendSculptor** when exceptions are actually taxonomy noise.

## Extension opportunities

- Auto-ticket creation for rule changes with risk scoring
- Link recovered leakage estimates to **Savings Tracker** baselines

## Differentiation

- **Spendscape** is strategic landscape storytelling on spend.
- **Spend Control Tower** is **control and compliance operations** over transactional flows.
